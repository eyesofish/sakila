好，我给你一个“从会用到接近实战”的完整练习清单。
不是玩具级 CRUD，而是按真实开发思路拆模块。

你照这个做一轮，MyBatis-Plus 基本算扎实。

---

第一阶段：基础打底（熟悉 MP 用法）

目标：把单表操作彻底熟练。

模块 1：电影管理（film）

你要实现：

1. 查询电影列表（分页）
2. 按标题模糊搜索
3. 按分类筛选
4. 按租赁价格区间筛选
5. 新增电影
6. 修改电影
7. 删除电影

你会练到：

- Page 分页
- QueryWrapper 条件拼接
- between
- like
- in
- 排序
- 基础 CRUD

做到这一层，你只是“会用”。

---

第二阶段：关系型思维（多表 + 真实业务）

目标：理解关系型数据库的价值。

模块 2：演员管理（actor + film_actor）

实现：

1. 查询演员列表
2. 查询某演员出演的电影
3. 给电影新增演员
4. 移除演员与电影的关联

你会练到：

- 多对多关系
- 中间表操作
- 自定义 SQL（XML 写 join）
- Service 层事务控制

---

模块 3：客户与租赁（customer + rental + inventory）

实现：

1. 查询客户列表（带最近租赁次数）
2. 查询某客户的租赁历史（分页）
3. 创建租赁记录（模拟下单）
4. 归还电影（更新 rental）

你会练到：

- 多表 join
- count 统计
- group by
- 事务
- 更新状态字段
- 乐观锁（可以自己加 version 字段练习）

---

第三阶段：统计与后台管理思维

目标：从“写接口”变成“做后台系统”。

模块 4：销售统计（payment）

实现：

1. 某时间段总收入
2. 每天收入趋势
3. 收入排名前 10 的电影
4. 某员工处理订单数统计

你会练到：

- sum
- group by
- order by
- 聚合查询
- DTO 映射
- 自定义 ResultMap

---

第四阶段：进阶（接近真实开发）

模块 5：后台管理改造

加上：

1. 统一返回结构（Result 封装）
2. 全局异常处理
3. 参数校验（@Valid）
4. 分页统一封装
5. 逻辑删除（自己加字段）
6. 自动填充创建时间 / 更新时间
7. 乐观锁插件

这一步才算“像样项目”。

---

第五阶段（可选加强）

如果你想更进一步：

1. 写接口测试（Postman 或 Swagger）
2. 用 MP 分页插件
3. 打开 SQL 日志分析
4. 加索引然后测试 explain
5. 模拟并发更新问题

- 用 Wrapper 拼复杂条件
- 写多表 join
- 写聚合统计
- 理解事务
- 理解外键
- 理解分页机制
- 理解 DTO 和实体分离

这才叫“能工作”。
